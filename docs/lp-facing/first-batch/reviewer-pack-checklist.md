# Reviewer Pack Checklist

Use this checklist before sending materials to first-batch reviewers.

## Send Before The Meeting

- One-page overview.
- Demo page screenshot.
- Sample completed report.
- Nontechnical fund manager reviewer pack, when the reviewer does not use
  GitHub.
- Reviewer-specific short message.
- Synthetic screenshots or sample report, if useful.
- Feedback form link or document.
- Clear meeting length: 30 minutes.

## Do Not Send

- Real LP confidential data.
- Real fund documents, side letters, bank details, tax records, or accounting
  records.
- Access codes inside GitHub.
- `.env` or `.env.local` files.
- Screenshots that reveal secrets, private settings, tokens, real reviewer
  identities, or private repository administration pages.
- GitHub repo access to nontechnical reviewers unless repo inspection is part of
  the review.

## Nontechnical Senior Reviewer Flow

Send:

- One-page overview.
- Demo page screenshot.
- Sample completed report.
- Nontechnical fund manager reviewer pack.
- Three focused questions.

Ask:

- What part is immediately understandable?
- What part would an LP or fund manager care about most?
- What would make this credible enough for a second conversation?

Avoid:

- Setup instructions.
- GitHub links.
- Code terminology.
- Internal roadmap detail unless they ask.

## Technical Reviewer Flow

Send:

- Private GitHub repo invitation.
- README setup instructions.
- `.env.example` explanation.
- Access-code owner workflow summary.

Ask:

- Did the setup work?
- Was the repo identity clear as Fund Governance Edition?
- Did the access-code mechanism make sense?
- Where did the demo feel fragile?

## Meeting Close

End each review by recording:

- Strongest value signal.
- Most confusing moment.
- Top missing feature.
- Whether the reviewer would join a second review.
- Whether the next step should be LP material, hosted demo, meeting pack,
  editable terms, or lifecycle realism.
